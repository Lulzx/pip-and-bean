# Architecture

How *Pip & Bean — The Big Slide Day* is put together, from script to MP4.

## The big picture

HyperFrames renders video by loading `index.html` in headless Chrome, **seeking** a
paused GSAP timeline to each frame time, and screenshotting. That single fact shapes
everything here: the picture at time *t* must be a **pure function of t** — no live
physics, no `Math.random()`, no `Date.now()`, no network fetches. Anything "random"
is seeded; anything "physical" is an analytic curve or a pre-authored tween.

```
script.json ──▶ gen_tts.py ──▶ assets/voices/*.mp3 + voice_manifest.json
                                        │
                                        ▼ (line start times laid out with gaps)
                               timeline.json
                                        │
                                        ▼
                               gen_index.py ──▶ index.html  (root orchestrator)
                                                    │
                 compositions/a1..a5.html ◀─────────┤ data-composition-src
                 lib/stage-kit.js         ◀─────────┤ <script> in <head>
                                                    ▼
                              npm run render ──▶ renders/*.mp4
```

## Root orchestrator — `index.html`

Generated by `gen_index.py` from `timeline.json`. It owns:

- **The five act mounts** — `<div data-composition-src="compositions/aN.html"
  data-start data-duration data-track-index="1">`. Each act is a self-contained
  sub-composition; the framework offsets its local timeline by `data-start`.
- **All audio** as `<audio>` clips with `data-start`/`data-duration`:

  | track | content | volume |
  |---|---|---|
  | 1  | the five act sub-compositions | — |
  | 10 | voice lines (52 clips, one per script line) | 0.97 narrator / 1.0 cast |
  | 11 | BGM: ukulele (acts 1–2) → sad piano (crush → act 3) → funshine (acts 4–5) | 0.13–0.16 |
  | 12 | SFX (chime, boing, pop, whoosh, thwack, win jingle) | per-event |
  | 13 | SFX overflow lane — catches boings that overlap track 12 | per-event |

- **Shared libraries** in `<head>`: GSAP, MotionPathPlugin, and `lib/stage-kit.js`.
  Sub-composition `<head>`s do **not** re-execute when mounted — anything an act
  needs at runtime must be loaded here. (Acts still carry their own `<script src>`
  tags for standalone viewing; they're inert in the mounted context.)

Timing constants (`timeline.json`):

| act | start | duration | content |
|---|---|---|---|
| a1 | 0.00 | 45.45 | title + meet Pip & Bean + the Great Big Slide |
| a2 | 45.45 | 50.97 | Milo arrives; push, grab, CRUNCH; the world dims |
| a3 | 96.42 | 64.44 | comfort → Milo alone → the insight → the invitation |
| a4 | 160.86 | 44.43 | the frown flips; building; slide train; sunshine |
| a5 | 205.29 | 48.91 | three magic moves; kindness line; outro + goodbyes |

Total: **254.2 s** at 1920×1080/30fps (≈7,626 frames).

## Acts — `compositions/a1..a5.html`

Each act is one HTML file with the same internal shape:

1. **CSS** — shared visual vocabulary per act (`.phase`, `.sky`, `.ground`, `.hill`,
   `.bubble` + `b-pip/b-bean/b-milo` variants, `.cap` narrator captions, `.badge`).
2. **Phases** — `<div class="phase" id="aN-pK">` full-frame scene containers,
   revealed/hidden by the timeline (iris wipes, pushes, crossfades).
3. **Inline `<script>`** —
   - `hfChar(kind, opts)` / `hfFace(opts)`: the cast as inline SVG (Pip, Bean, Milo
     with parametric mouths, brows, look direction, blush).
   - `talk / blink / bubble / caption` tween helpers, synced to voice line times.
   - `HFKIT.dress("aN")` — procedural scenery + prop generation (see
     [STAGE-KIT.md](STAGE-KIT.md)).
   - one paused GSAP timeline with all choreography, registered on
     `window.__timelines["aN"]`.
   - `HFKIT.wind(tl, "aN", DUR)` — seek-safe wind sway, added just before
     registration.

Character/bubble/caption timings are hand-synced to the voice line start times from
`timeline.json` (each act works in local time: `global start − act start`).

## Fonts

`inject_fonts.py` embeds Baloo 2 (400/800) and Patrick Hand as base64 data-URI
`@font-face` blocks directly into each act (idempotent, marked with
`data-hf-fonts`). No network fetch at render time; fonts are always ready when the
first frame is captured.

## Determinism rules (the contract everything obeys)

- One **paused** timeline per composition, registered on `window.__timelines`.
- Every timed element: `data-start`, `data-duration`, `data-track-index`, and
  `class="clip"`.
- No `Math.random` / `Date.now` / network. Seeded PRNG only (`HFKIT.rng`).
- Media playback is framework-owned; `<audio>` clips declare timing, never autoplay.
